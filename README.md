### 网络线路

**去程路由**：
- 电信/联通：各自骨干网到美西，转GTT到目标机房
- 移动：CMI到美西，转Cogent

**回程路由**：
- 电信：Telia到美西转电信

- 洛杉矶：186ms
- 水牛城/纽约：220-250ms
- 芝加哥：200ms左右

ColoCrossing提供100%网络在线时间SLA。但实际使用中，有用户反馈偶尔会遇到：
- 节点故障（主要是2023-2024年黑五促销期间的特价机型）
- IP被墙（如果做了违规操作）
- 晚高峰速度下降

不过作为机房自营的服务，稳定性整体还是比很多小商家要好。毕竟ColoCrossing已经运营了20多年，给很多知名商家提供上游服务。

### 2. IP被墙了怎么办？

ColoCrossing不提供免费换IP服务。如果IP被墙：
- 可以尝试提交工单申请更换（可能收费）
- 或者使用CDN等方式绕过

所以建议：
- 不要用来做违规业务
- 定期备份数据
- 如果主要面向中国用户，考虑加CDN

### 3. 能装Windows吗？

可以。VPS和Cloud Metal都支持Windows Server 2012-2025。但需要注意：
- Windows需要更多资源，建议至少2GB内存
- 可能需要额外付费（具体价格咨询客服）

独立服务器可以装任何系统，包括Windows Server、各种Linux发行版、甚至ESXi虚拟化平台。

### 4. 流量真的不限吗？

独立服务器是限制40TB/月，但根据老用户反馈，只要不是明显滥用（比如24小时跑满带宽做BT种子服务器），超一些问题不大。

部分VPS套餐标注"20TB/月流量"，但也有人反馈后台实际显示330TB限制。这个可能是ColoCrossing为了防止滥用设置的软限制。

总之，正常使用肯定够了。如果你真的需要跑超大流量，建议：
- 提前跟客服沟通确认
- 选择独立服务器而不是VPS
- 考虑购买更高级别的服务

### 5. 哪个机房到中国速度最好？

根据测评数据：
- **洛杉矶**：平均延迟最低（186ms），但晚高峰可能拥堵
- **芝加哥**：延迟适中（200ms左右），相对稳定
- **水牛城/纽约**：延迟较高（220-250ms），适合美东业务

如果主要面向中国用户，建议优先选洛杉矶，其次芝加哥。

### 6. 能做什么，不能做什么？

**可以做的**：
- 个人博客、网站托管
- 学习Linux运维
- 搭建开发测试环境
- 数据存储和备份
- PT下载（不要太过分）

**不建议/禁止的**：

- 侵权内容托管

总之，合法合规使用，一般不会有问题。
